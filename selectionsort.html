Selectionsort

Algoritmo di ordinamento per selezione

Introduzione

Selectionsort divide l'array in una parte ordinata all'inizio e una disordinata alla fine. In ogni iterazione, trova il minimo nella parte disordinata e lo scambia con il primo elemento disordinato.

Complessità temporale: O(n²) in tutti i casi

Complessità spaziale: O(1) (ordinamento in loco)

Come Funziona

  1. Trova l'elemento minimo nell'array non ordinato
  2. Scambialo con il primo elemento non ordinato
  3. Ripeti per la parte rimanente dell'array

Implementazione in C

void selectionSort(int arr[], int n) {
    int i, j, min_idx;

    // Uno alla volta sposta il limite dell'array non ordinato
    for (i = 0; i < n-1; i++) {
        // Trova l'elemento minimo nell'array non ordinato
        min_idx = i;
        for (j = i+1; j < n; j++)
            if (arr[j] < arr[min_idx])
                min_idx = j;

        // Scambia l'elemento minimo trovato con il primo elemento
        if (min_idx != i) {
            int temp = arr[min_idx];
            arr[min_idx] = arr[i];
            arr[i] = temp;
        }
    }
}

L'algoritmo effettua sempre lo stesso numero di confronti, indipendentemente dall'ordine iniziale dell'array.

Vantaggi e Svantaggi

Vantaggi:

  • Semplice da implementare
  • Numero di scambi minimo (al massimo n-1)
  • In loco e non richiede memoria aggiuntiva

Svantaggi:

  • Sempre O(n²), anche per array quasi ordinati
  • Non stabile (può cambiare l'ordine relativo di elementi uguali)
  • Non adatto per grandi array

Confronto con Insertionsort

Selectionsort effettua meno scambi di Insertionsort (massimo n-1 vs fino a n²/2), ma più confronti. Insertionsort è generalmente preferibile per array piccoli.

Conclusione

Selectionsort è utile quando il costo degli scambi è alto rispetto ai confronti, ma per la maggior parte delle applicazioni moderne, algoritmi più efficienti sono preferibili.
