Insertionsort

Algoritmo di ordinamento per inserimento

Introduzione

Insertionsort costruisce l'array ordinato finale un elemento alla volta. È molto efficiente per array piccoli o quasi ordinati, e viene spesso usato come base per algoritmi più complessi.

Complessità temporale: O(n²) nel caso peggiore, O(n) nel caso migliore

Complessità spaziale: O(1) (ordinamento in loco)

Come Funziona

  1. Considera il primo elemento come già ordinato
  2. Per ogni elemento successivo, inseriscilo nella posizione corretta nella parte già ordinata
  3. Sposta gli elementi più grandi a destra per fare spazio

Implementazione in C

void insertionSort(int arr[], int n) {
    int i, key, j;
    for (i = 1; i < n; i++) {
        key = arr[i];
        j = i - 1;

        // Sposta gli elementi dell'array ordinato che sono
        // maggiori della chiave di una posizione avanti
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j = j - 1;
        }
        arr[j + 1] = key;
    }
}

L'algoritmo mantiene una parte sinistra ordinata e inserisce ogni nuovo elemento nella posizione corretta.

Vantaggi e Svantaggi

Vantaggi:

  • Efficiente per array piccoli (n ≤ 10-20)
  • Molto efficiente per array quasi ordinati
  • Stabile e in loco
  • Semplice da implementare

Svantaggi:

  • O(n²) per array grandi o inversamente ordinati
  • Non adatto per grandi dataset

Conclusione

Insertionsort è eccellente per piccoli array o come parte di algoritmi ibridi. È spesso usato in combinazione con algoritmi più veloci per ottimizzare le prestazioni.
