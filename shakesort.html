Shakesort (Cocktail Sort)

Variante bidirezionale di Bubblesort

Introduzione

Shakesort, anche conosciuto come Cocktail Sort, è una variante bidirezionale di Bubblesort. Invece di muoversi solo da sinistra a destra, alterna le direzioni in ogni passaggio, riducendo leggermente il numero di confronti.

Complessità temporale: O(n²) in tutti i casi

Complessità spaziale: O(1) (ordinamento in loco)

Come Funziona

  1. Passaggio da sinistra a destra: sposta il massimo alla fine
  2. Passaggio da destra a sinistra: sposta il minimo all'inizio
  3. Ripeti alternando le direzioni fino a quando non ci sono scambi

Implementazione in C

void cocktailSort(int a[], int n) {
    int swapped = 1;
    int start = 0;
    int end = n - 1;

    while (swapped) {
        swapped = 0;

        // Passaggio da sinistra a destra
        for (int i = start; i < end; ++i) {
            if (a[i] > a[i + 1]) {
                int temp = a[i];
                a[i] = a[i + 1];
                a[i + 1] = temp;
                swapped = 1;
            }
        }

        // Se non ci sono stati scambi, l'array è ordinato
        if (!swapped)
            break;

        swapped = 0;
        --end;

        // Passaggio da destra a sinistra
        for (int i = end - 1; i >= start; --i) {
            if (a[i] > a[i + 1]) {
                int temp = a[i];
                a[i] = a[i + 1];
                a[i + 1] = temp;
                swapped = 1;
            }
        }

        ++start;
    }
}

Questa implementazione ottimizzata ferma l'algoritmo quando non ci sono più scambi in un passaggio completo.

Vantaggi rispetto a Bubblesort

  • Leggermente più efficiente in pratica
  • Stesso numero di scambi ma meno confronti inutili
  • Utile per array quasi ordinati

Conclusione

Shakesort è un miglioramento marginale di Bubblesort, ancora O(n²) ma con prestazioni leggermente migliori. È utile principalmente per scopi didattici.
