GnomeSort

Algoritmo di ordinamento ottimo per piccoli array

Introduzione

GnomeSort è un algoritmo di ordinamento molto semplice che funziona in modo simile al sort “gnome” di cui è stato così chiamato per il suo comportamento quasi “dinosauro”. Confronta e, se necessario, scambia coppie di elementi adiacenti fino a quando l’intero array non è ordinato. È ideale per array di dimensioni ridotte.

Complessità temporale: O(n²) in tutti i casi

Complessità spaziale: O(1) (ordinamento in loco)

Come Funziona

  1. Si parte dall’indice 0.
  2. Se l’indice è 0 o l’elemento corrente è maggiore o uguale al precedente, si passa al successivo.
  3. Altrimenti si scambiano gli elementi e si torna indietro di una posizione.
  4. Il processo si ripete finché non si raggiunge la fine dell’array.

Implementazione di Esempio (C)


#include <stdio.h>
#include <stdlib.h>
#include <time.h>

const int GRANDEZZA = 10;
const int MAX = 100;
const int MIN = 1;

void riempiArr (int arr[GRANDEZZA]) {
    srand(time(NULL));
    for (int i = 0; i < GRANDEZZA; i++){
        arr[i] = (rand() % (MAX - MIN + 1)) + MIN;
    }
}

void gnomeSort (int arr[GRANDEZZA]) {
    int i = 0;
    while (i < GRANDEZZA) {
        if (i == 0 || arr[i] >= arr[i-1]) {
            i++;
        } else {
            int temp = arr[i];
            arr[i] = arr[i-1];
            arr[i-1] = temp;
            i--;
        }
    }
}

void stampaArr(int arr[GRANDEZZA]) {
    for(int i = 0; i < GRANDEZZA; i++){
        printf("%d\t", arr[i]);
    }
}

int main () {
    int arr[GRANDEZZA];
    printf("\n---\tINIZIO\t---\n");

    riempiArr(arr);
    printf("\nArray non ordinato:\n");
    stampaArr(arr);
    gnomeSort(arr);
    printf("\nArray ordinato:\n");
    stampaArr(arr);

    printf("\n---\nFINE\t---\n");
    return 0;
}

Il ciclo principale utilizza una singola variabile di indice che avanza solo quando l’array è già ordinato localmente, garantendo un comportamento simile a quello di un “gnomo” che si muove lentamente, fermandosi a controllare la propria posizione.

Ottimizzazioni

Una versione ottimizzata può terminare l’algoritmo non appena l’intero array è ordinato, evitando inutili passaggi:


void gnomeSortOptimized(int arr[GRANDEZZA]) {
    int i = 0;
    int sorted = 0;  // flag che indica se l’array è già ordinato
    while (i < GRANDEZZA) {
        if (i == 0 || arr[i] >= arr[i-1]) {
            i++;
            sorted = 1;   // nessuno scambio nella posizione corrente
        } else {
            int temp = arr[i];
            arr[i] = arr[i-1];
            arr[i-1] = temp;
            i--;
            sorted = 0;   // scambio avvenuto, quindi non è ancora ordinato
        }
    }
    // Se l’algoritmo è finito con sorted==1, l’array era già ordinato
}

Vantaggi e Svantaggi

Vantaggi:

  • Semplice da implementare e comprendere
  • Ordinamento in loco
  • Stabile (preserva l’ordine degli elementi uguali)

Svantaggi:

  • Molto lento per array grandi (O(n²))
  • Non adatto per uso pratico in applicazioni di produzione

Conclusione

GnomeSort è un algoritmo didattico eccellente che illustra concetti di confronto e scambio in modo intuitivo. È utile soprattutto per piccoli dataset e per introdurre principi di ordinamento. Per dataset più grandi, è consigliabile optare per algoritmi più efficienti come Quicksort, MergeSort o HeapSort.
